## 第八章：高级特性

### 章节引言

Hermes 提供了丰富的扩展功能，让 AI Agent 能够语音交互、提供 API 服务、集成外部工具生态，并支持 RL 强化学习训练。本章将介绍语音模式、API 服务器、MCP 集成和 RL 训练四大高级特性。通过学习，你将能够构建语音助手、将 Hermes 集成到外部应用、扩展工具能力，以及优化 AI 模型。前置知识包括第四章消息网关和第六章自动化。

#### 8.1.2 AudioRecorder 核心参数

**源码**：`tools/voice_mode.py` 第 373 行

```python
class AudioRecorder:
    """音频录制器，支持静音自动停止"""

    supports_silence_autostop = True

    # 录音参数
    _sample_rate = 16000        # Whisper 原生采样率
    _channels = 1                # 单声道
    _dtype = "int16"             # 16位 PCM
    _sample_width = 2            # 每样本 2 字节

    # 静音检测
    _silence_threshold = 200     # RMS 阈值 (int16 范围 0-32767)
    _silence_duration = 3.0      # 静音持续秒数后自动停止
    _max_wait = 15.0            # 最长等待语音时间(秒)
```

### 8.4 MCP 集成

#### 8.4.1 MCP 协议概述

**源码**：`tools/mcp_tool.py`

Model Context Protocol (MCP) 是 Anthropic 提出的工具协议标准：

```
┌─────────────┐     MCP      ┌─────────────┐
│   Client    │◄────────────►│   Server    │
│  (Hermes)   │              │ (Filesystem │
│             │              │  Git, etc.) │
└─────────────┘              └─────────────┘

MCP 核心概念：
- Resources: 可读资源（文件、数据）
- Tools: 可执行操作
- Prompts: 预定义提示模板
```

#### 8.4.2 MCP 配置格式

```yaml
# config.yaml
mcp:
  enabled: true

  # Stdio 传输（命令 + 参数）
  servers:
    filesystem:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
      env: {}
      timeout: 120         # 工具调用超时(秒)
      connect_timeout: 60  # 初始连接超时

    github:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_PERSONAL_ACCESS_TOKEN: ${GITHUB_TOKEN}

    slack:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-slack"]
      env:
        SLACK_BOT_TOKEN: ${SLACK_BOT_TOKEN}
        SLACK_TEAM_ID: ${SLACK_TEAM_ID}

    brave-search:
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-brave-search"]
      env:
        BRAVE_API_KEY: ${BRAVE_API_KEY}

  # HTTP/StreamableHTTP 传输
  remote_api:
    url: "https://my-mcp-server.example.com/mcp"
    headers:
      Authorization: "Bearer sk-..."
    timeout: 180
```

#### 8.4.3 传输类型

| 传输类型 | 配置方式 | 适用场景 |
|----------|----------|----------|
| **Stdio** | `command` + `args` | 本地 MCP 服务器 |
| **HTTP** | `url` + `headers` | 远程 MCP 服务器 |
| **StreamableHTTP** | `url` + `headers` | 支持流式的远程服务器 |

#### 8.4.4 MCP 采样配置

MCP 服务器可以请求 LLM 完成复杂任务：

```yaml
mcp:
  servers:
    analysis:
      command: "python"
      args: ["/path/to/analysis_server.py"]

      # 采样配置（可选）
      sampling:
        enabled: true              # 默认启用
        model: "gemini-3-flash"  # 覆盖默认模型
        max_tokens_cap: 4096      # 最大 token
        timeout: 30               # LLM 调用超时
        max_rpm: 10              # 每分钟最大请求
        allowed_models: []        # 白名单（空=全部允许）
        max_tool_rounds: 5       # 工具循环限制
        log_level: "info"         # 审计日志级别
```

#### 8.4.6 FastMCP 开发

使用 FastMCP 快速开发自定义 MCP 服务器：

```python
# server.py
from fastmcp import FastMCP

mcp = FastMCP("My Server")

@mcp.tool()
def search_docs(query: str) -> str:
    """搜索项目文档"""
    return f"搜索 {query} 的结果..."

@mcp.tool()
def get_file(path: str) -> str:
    """读取文件内容"""
    with open(path) as f:
        return f.read()

@mcp.resource("docs://{path}")
def get_doc(path: str) -> str:
    """获取文档内容"""
    with open(path) as f:
        return f.read()

@mcp.prompt()
def code_review(repo: str) -> str:
    """代码审查提示"""
    return f"审查代码仓库 {repo} 的最新提交"

mcp.run()
```

**运行自定义服务器：**

```yaml
mcp:
  servers:
    my-server:
      command: python
      args: ["/path/to/server.py"]
      env:
        DEBUG: "true"
```

### 8.5 RL 训练

#### 8.5.1 RL 系统架构

**源码**：`tools/rl_training_tool.py`

Hermes 与 Tinker-Atropos RL 系统集成：

```
┌─────────────────────────────────────────────────────────────┐
│                   RL Training System                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────┐     ┌──────────────┐     ┌──────────────┐ │
│  │  Hermes CLI │────►│  Atropos API │────►│ Tinker       │ │
│  │  (rl_* 工具) │     │  (run-api)   │     │ (训练器)      │ │
│  └─────────────┘     └──────────────┘     └──────────────┘ │
│                            │                       │         │
│                            ▼                       ▼         │
│                     ┌──────────────┐     ┌──────────────┐  │
│                     │  Inference   │     │  Environment │  │
│                     │  Server     │     │  (任务环境)   │  │
│                     └──────────────┘     └──────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 8.5.2 环境变量

#### 8.5.3 RL 工具接口

```python
# 1. 列出可用环境
await rl_list_environments() -> JSON
# 扫描 tinker-atropos/tinker_atropos/environments/ 目录

# 2. 选择环境
await rl_select_environment(name: str) -> JSON

# 3. 获取当前配置
await rl_get_current_config() -> JSON
# 返回: configurable_fields (可修改) + locked_fields (锁定)

# 4. 编辑配置
await rl_edit_config(field: str, value: Any) -> JSON

# 5. 启动训练
await rl_start_training() -> JSON
# 启动三个进程: run-api, launch_training.py, environment.py

# 6. 检查状态
await rl_check_status() -> JSON
# 状态: pending, starting, running, stopping, stopped, completed, failed

# 7. 停止训练
await rl_stop_training(run_id: str) -> JSON

# 8. 获取结果
await rl_get_results(run_id: str) -> JSON
# 返回 WandB 指标
```
