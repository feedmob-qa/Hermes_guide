## 0.3 五分钟上手

### 0.3.1 前置要求

| 类型 | 要求 |
|------|------|
| 最低 | Python >= 3.10, pip, 网络连接 |
| 推荐 | 8GB+ RAM, Linux/macOS/WSL2, API Key |

### 0.3.2 一键安装

```bash
# Linux / macOS / WSL2
curl -fsSL https://get.hermes-agent.dev | bash

# 或使用 pip
pip install hermes-agent

# Termux (Android)
pkg install hermes-agent
```

### 0.3.3 首次配置

```bash
# 运行设置向导
hermes setup

# 或手动配置 API Key
export OPENROUTER_API_KEY="sk-or-v1-xxxxx"

# 验证安装
hermes doctor
```

### 0.3.4 开始使用

```bash
# 启动交互式 TUI
hermes

# 或单次对话
hermes "你好，请介绍一下你自己"
```

## 附录

### 附录A：术语表

#### A.1 核心术语

| 术语 | 英文 | 定义 | 源码位置 |
|------|------|------|----------|
| Agent Loop | Agent Loop | AI Agent 的核心循环：感知→思考→行动→反馈 | `run_agent.py` |
| Tool | Tool | 可被 LLM 调用的函数，具备 schema 和 handler | `tools/registry.py` |
| Toolset | Toolset | 工具分组，包含同一功能域的工具集合 | `toolsets.py` |
| Skill | Skill | 封装的工作流，以 SKILL.md 格式定义 | `skills/` |
| Gateway | Gateway | 消息网关，连接 Hermes 与各消息平台 | `gateway/` |
| Platform Adapter | Platform Adapter | 平台适配器，实现特定平台的协议转换 | `gateway/adapters/` |
| MCP | Model Context Protocol | Anthropic 提出的工具协议标准 | `tools/mcp_tool.py` |
| Tool Registry | Tool Registry | 工具注册中心，负责工具发现和调度 | `tools/registry.py` |
| Self-Registration | Self-Registration | 工具模块导入时自动注册到 Registry | `tools/*.py` |
| Session | Session | 单个对话会话，包含消息历史 | `sessions/` |

#### A.2 网关与平台术语

| 术语 | 定义 | 支持平台 |
|------|------|----------|
| Webhook | 平台推送消息到 Hermes 的 HTTP 回调 | 所有支持 Webhook 的平台 |
| Long Polling | 客户端保持长连接拉取消息 | Telegram, Discord, Slack |
| Bot Token | 平台为 Bot 分配的访问令牌 | Telegram, Discord, Slack |
| OAuth Flow | 用户授权第三方应用访问的流程 | Nous Portal, Google Gemini, DeepSeek |
| Platform Adapter | 将平台特有协议转换为统一消息格式的适配器 | 所有平台 |
| Cross-Platform | 跨平台功能，支持在多个平台间同步消息 | - |

#### A.3 工具与技能术语

| 术语 | 定义 | 示例 |
|------|------|------|
| Tool Schema | 工具的 OpenAI 格式函数定义 | `{"name": "web_search", "parameters": {...}}` |
| Tool Handler | 工具的实际执行函数 | `def web_search(args):` |
| check_fn | 工具可用性检查函数 | `def check_ollama(): return shutil.which("ollama")` |
| Toolset Alias | 工具集的别名 | `safe` → `hermes-safemode` |
| Skill Trigger | 技能触发条件 | 自然语言匹配、正则、关键词 |
| Memory Provider | 记忆提供者插件接口 | Honcho, Supermemory |
| LMWiki | LLM 原生知识库 | 三层架构：Raw/Wiki/SCHEMA |

#### A.4 模型与推理术语

| 术语 | 定义 | 相关配置 |
|------|------|----------|
| Provider | 模型供应商 | `config.yaml` providers |
| Model Routing | 模型路由策略 | `provider_routing_strategy` |
| Fallback Model | 回退模型配置 | `fallback_models` |
| Context Budget | 上下文预算管理 | `max_context_tokens` |
| Tool Call Parser | 工具调用解析器 | `--jinja`, `--tool-call-parser` |
| Streaming | 流式输出 | SSE, Block Chunker |
| Token | Token 计数单位 | tiktoken, cl100k_base |

#### A.5 运维与安全术语

| 术语 | 定义 | 相关命令 |
|------|------|----------|
| YOLO Mode | 绕过命令审批的模式 | `/yolo` |
| Approval Queue | 待审批命令队列 | `/approve` `/deny` |
| Backpressure | 背压机制，防止过载 | 排队、超时 |
| Idempotency | 幂等性控制 | 防重放 |
| Sandbox | Docker 沙箱隔离 | `tools/environments/` |
| MCP OAuth | MCP 服务器认证 | `tools/mcp_oauth.py` |

| E005 | `Context overflow` | 上下文溢出 | 使用 `/compress` 压缩 |
| E006 | `Tool execution failed` | 工具执行失败 | 查看工具日志 |
| E007 | `Platform auth failed` | 平台认证失败 | 检查 Bot Token |
| E008 | `Gateway not running` | 网关未启动 | `hermes gateway start` |
| E009 | `Sandbox error` | 沙箱错误 | 检查 Docker 配置 |
| E010 | `MCP server error` | MCP 服务器错误 | 检查 MCP 配置 |
| E011 | `Memory DB locked` | 内存库锁定 | 重启 hermes |
| E012 | `Skill not found` | 技能未找到 | `hermes skills install` |
| E013 | `Voice engine error` | 语音引擎错误 | 检查 TTS/STT 配置 |
| E014 | `OAuth token expired` | OAuth 令牌过期 | 重新授权 |
| E015 | `Device not approved` | 设备未批准 | `/approve-device` |

#### D.3 冷却梯度策略

```
冷却梯度 (Cooldown Gradient)
├── 即时重试 (0-5秒)
│   └── 适用: 网络抖动、临时超时
├── 短等待 (30秒)
│   └── 适用: API 限流
├── 中等待 (5分钟)
│   └── 适用: 速率限制
├── 长等待 (30分钟)
│   └── 适用: 账户限额
└── 指数退避 (1m → 2m → 4m → 8m → 16m)
    └── 适用: 持续限流
```

#### D.5 常见场景排障

**场景 1: 消息不响应**
```
诊断步骤:
1. 检查平台连接
   hermes platform <platform> --test

2. 检查网关状态
   hermes gateway status

3. 检查日志
   hermes logs --filter "message.*received"

4. 检查审批队列
   hermes approval list
```

**场景 2: 模型调用失败**
```
诊断步骤:
1. 验证 API Key
   echo $OPENROUTER_API_KEY

2. 测试 API 连通性
   curl -H "Authorization: Bearer $OPENROUTER_API_KEY" \
        https://openrouter.ai/api/v1/models

3. 检查模型配置
   hermes model list

4. 切换回退模型
   hermes model <fallback-model>
```

**场景 3: 工具被拒绝**
```
诊断步骤:
1. 检查工具状态
   hermes tools --list

2. 检查审批状态
   hermes approval list

3. 检查黑名单
   hermes config get security.blacklist

4. 临时放行
   hermes yolo on  # 谨慎使用
```

**场景 4: 上下文溢出**
```
诊断步骤:
1. 检查上下文使用
   hermes stats --context

2. 压缩上下文
   /compress

3. 调整阈值
   hermes config set model.context_compression_threshold 100000

4. 切换更大上下文模型
   hermes model claude-3-200k
```

**场景 5: WSL2 网络问题**
```
诊断步骤:
1. 检查 WSL2 状态
   wsl.exe -l -v

2. 测试 localhost 连通性
   curl http://localhost:11434

3. 配置 Windows 防火墙
   netsh advfirewall firewall add rule name="Hermes Ollama" ^
     dir=in action=allow protocol=tcp localport=11434

4. 使用 Windows IP
   修改 config.yaml:
   providers:
     ollama:
       base_url: http://$(hostname -I | awk '{print $1}'):11434
```

**成本计算公式:**

```
日成本 = Σ(模型调用次数 × 模型单价 × 平均 Token 数 / 1000)

示例:
- Claude 3.5 Sonnet: $3/1M input tokens
- 日均 1000 次对话
- 每次平均 5000 input tokens

日成本 = 1000 × ($3/1M) × (5000/1000)
       = 1000 × $0.000003 × 5
       = $0.015/天
```
