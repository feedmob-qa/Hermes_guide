## 第六章：自动化与终端后端

### 章节引言

Hermes 不仅是一个响应式 AI 助手，更是一个能够主动执行任务的自动化引擎。本章将介绍 Cron 调度、委托任务、终端后端和命令安全四大核心能力。通过学习，你将能够设置定时任务、自动执行重复性工作、隔离危险命令，以及构建复杂的并行处理流程。前置知识包括第三章的工具系统和第二章的配置体系。

---

### 6.1 Cron 调度

#### 6.1.1 定时任务架构

Hermes 的调度系统由三个核心组件构成：

```bash
┌─────────────────────────────────────────────────┐
│              Cron Scheduler                      │
├─────────────────────────────────────────────────┤
│  Job Store (JSON)  │  Tick Loop  │  Executor    │
│  ~/.hermes/cron.json  每60s检查  →  Agent Loop  │
└─────────────────────────────────────────────────┘
```

#### 6.1.2 cronjob 工具详解

**源码**：`tools/cronjob_tools.py`

```python
def cronjob(
    action: str,              # 'create' | 'list' | 'update' | 'pause' | 'resume' | 'remove' | 'run'
    job_id: Optional[str] = None,
    prompt: Optional[str] = None,
    schedule: Optional[str] = None,  # '30m', 'every 2h', '0 9 * * *', ISO timestamp
    name: Optional[str] = None,
    repeat: Optional[int] = None,   # 重复次数
    deliver: Optional[str] = None,   # 'local' | 'origin' | 'platform:chat_id:thread_id'
    include_disabled: bool = False,
    skill: Optional[str] = None,
    skills: Optional[List[str]] = None,
    model: Optional[Dict] = None,   # {'provider': str, 'model': str}
    provider: Optional[str] = None,
    base_url: Optional[str] = None,
    reason: Optional[str] = None,
    script: Optional[str] = None,   # 预执行脚本路径
)
```

#### 6.1.3 调度格式

Hermes 支持多种调度格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 自然语言 | `每天9点`、`每周一10点`、`每隔30分钟` | 自动转换为 cron 表达式 |
| 相对时间 | `30m`、`2h`、`in 1 hour` | 一次性任务 |
| 标准 cron | `0 9 * * *` | 5字段标准格式 |
| ISO 时间戳 | `2024-12-25T09:00:00` | 指定时间执行一次 |

#### 6.1.4 自然语言创建示例

```bash
# 创建每日简报任务
/cron create "每个工作日早上8点发送今日计划" --name daily-plan --schedule "0 8 * * 1-5"

# 创建每周报告
/cron create "每周一早上10点总结上周工作" --name weekly-summary --schedule "0 10 * * 1"

# 创建一次性提醒
/cron create "2小时后提醒开会" --schedule "2h"

# 创建每月任务
/cron create "每月1号凌晨整理上月支出报表" --name monthly-expense --schedule "0 0 1 * *"
```

#### 6.1.5 多平台投递

```bash
# 发送到当前平台
/cron create "每天18点发送项目进度" --deliver origin

# 指定 Telegram 聊天
/cron create "每天发送报告" --deliver telegram:123456789

# 指定 Discord 频道
/cron create "每天发送报告" --deliver discord:987654321:123456789

# 本地执行（不发消息）
/cron create "凌晨备份数据" --deliver local
```

#### 6.1.6 自动化模板

```yaml
# 预置模板
templates:
  daily-plan:
    schedule: "0 8 * * 1-5"
    prompt: "生成今日工作计划，列出3个优先事项"
    deliver: origin

  news-digest:
    schedule: "0 9 * * *"
    prompt: "获取最新科技新闻，整理摘要"
    deliver: origin

  pr-review:
    schedule: "0 17 * * 5"
    prompt: "检查 GitHub Pull Requests，标记需要关注的项目"
    skills: [github]
    deliver: origin

  backup:
    schedule: "0 2 * * *"
    prompt: "执行数据库备份并验证完整性"
    deliver: local
```

#### 6.1.7 安全扫描

`cronjob_tools.py` 中实现了 `_scan_cron_prompt()` 函数，防止定时任务中的恶意操作：

```python
DANGEROUS_PATTERNS = [
    # Prompt 注入
    (r'ignore.*instructions', 'prompt_injection'),
    (r'disregard.*rules', 'prompt_injection'),

    # 信息泄露
    (r'curl.*\$(KEY|TOKEN|SECRET)', 'credential_exposure'),
    (r'wget.*\$(KEY|TOKEN|SECRET)', 'credential_exposure'),
    (r'cat.*\.env', 'credential_exposure'),

    # SSH 后门
    (r'authorized_keys', 'ssh_backdoor'),
    (r'/etc/sudoers', 'privilege_escalation'),

    # 破坏性操作
    (r'rm\s+-rf\s+/', 'destructive_delete'),
]
```

### 6.2 委托与并行

#### 6.2.1 delegate_task 工具详解

**源码**：`tools/delegate_tool.py`

```python
def delegate_task(
    goal: Optional[str] = None,           # 单任务目标
    context: Optional[str] = None,        # 上下文信息
    toolsets: Optional[List[str]] = None,  # 可用工具集
    tasks: Optional[List[Dict]] = None,    # 批量任务
    max_iterations: Optional[int] = None, # 单子代理最大迭代
    acp_command: Optional[str] = None,     # ACP 子进程传输命令
    acp_args: Optional[List[str]] = None,  # ACP 参数
    role: Optional[str] = None,           # 'leaf' | 'orchestrator'
    parent_agent=None,
)
```

#### 6.2.2 角色系统

| 角色 | 说明 | 可委托 |
|------|------|--------|
| `leaf` | 专注工作者，不能进一步委托 | ❌ |
| `orchestrator` | 编排器，可以创建子工作进程 | ✅ |

#### 6.2.3 深度限制

```python
MAX_DEPTH = 1  # 扁平结构: 父(0) -> 子(1)
_MAX_SPAWN_DEPTH_CAP = 3  # 最大嵌套深度
```

#### 6.2.4 默认阻止的工具

```python
DELEGATE_BLOCKED_TOOLS = frozenset([
    "delegate_task",   # 禁止递归委托
    "clarify",         # 禁止用户交互
    "memory",          # 禁止写入共享 MEMORY.md
    "send_message",    # 禁止跨平台副作用
    "execute_code",    # 子代理应推理而非写脚本
])
```

#### 6.2.5 使用示例

**单任务委托：**

```bash
用户: 分析 /tmp/logs 目录下的错误日志

Hermes 使用 delegate_task:
goal='分析错误日志并总结问题'
context='日志目录: /tmp/logs\n项目: production-api'
toolsets=['terminal', 'file']
```

**批量并行任务：**

```bash
用户: 并行分析这100个文件

Hermes 使用 delegate_task:
tasks=[
    {"goal": "分析文件 1-20", "toolsets": ["file", "terminal"]},
    {"goal": "分析文件 21-40", "toolsets": ["file", "terminal"]},
    {"goal": "分析文件 41-60", "toolsets": ["file", "terminal"]},
    {"goal": "分析文件 61-80", "toolsets": ["file", "terminal"]},
    {"goal": "分析文件 81-100", "toolsets": ["file", "terminal"]},
]
# 5个子代理并行执行
# 主代理汇总结果
```

**编排器模式：**

```bash
用户: 重构整个微服务架构

Hermes 使用 delegate_task:
goal='重构微服务架构'
role='orchestrator'
context='项目: 电商平台\n规模: 12个微服务'
# 编排器可以创建更多子代理
```

#### 6.2.6 配置选项

```yaml
# config.yaml
delegation:
  max_concurrent_children: 3      # 最大并发子代理数
  child_timeout_seconds: 300       # 子代理超时 (秒)
  max_spawn_depth: 1               # 嵌套深度
  orchestrator_enabled: true       # 编排器角色开关
  max_iterations: 50               # 默认最大迭代
  model: "claude-sonnet-4"        # 子代理专用模型
  provider: "openrouter"           # 子代理专用提供者
```

### 6.3 终端后端

#### 6.3.1 终端架构

Hermes 通过环境变量 `TERMINAL_ENV` 支持多种执行后端：

```bash
┌─────────────────────────────────────────┐
│          Hermes Agent                    │
├─────────────────────────────────────────┤
│       TERMINAL_ENV 环境变量              │
├─────────┬─────────┬─────────┬────────────┤
│  local  │ docker  │  ssh    │   modal    │
│ (本地)   │ (容器)  │ (远程)  │  (云沙箱)  │
└─────────┴─────────┴─────────┴────────────┘
```

#### 6.3.2 terminal_tool 工具详解

**源码**：`tools/terminal_tool.py`

```python
def terminal_tool(
    command: str,                      # 要执行的命令
    background: bool = False,           # 后台执行
    timeout: Optional[int] = None,      # 超时秒数
    task_id: Optional[str] = None,      # 环境隔离标识
    force: bool = False,                # 跳过危险命令检查
    workdir: Optional[str] = None,      # 工作目录
    pty: bool = False,                  # 伪终端模式
    notify_on_complete: bool = False,  # 完成通知
    watch_patterns: Optional[List[str]] = None,  # 输出监控模式
)
```

#### 6.3.3 环境配置变量

| 环境 | 变量 | 说明 |

| **基础** | `TERMINAL_LIFETIME_SECONDS` | 环境生命周期 |

### 6.4 命令安全

#### 6.4.1 approval.py 核心函数

**源码**：`tools/approval.py`

```python
def check_all_command_guards(
    command: str,              # 要执行的命令
    env_type: str,              # 'local', 'ssh', 'docker', 等
    approval_callback=None,   # CLI 交互回调
) -> dict
```

#### 6.4.2 危险命令模式

```python
DANGEROUS_PATTERNS = [
    # 文件系统危险操作
    (r'\brm\s+(-[^\s]*\s+)*/', "delete in root path"),
    (r'\brm\s+-[^\s]*r', "recursive delete"),
    (r'\bchmod\s+.*777', "world/other-writable permissions"),
    (r'\bchown\s+.*R\s+root', "recursive chown to root"),
    (r'\bmkfs\b', "format filesystem"),

    # SQL 操作
    (r'\bDROP\s+TABLE\b', "SQL DROP"),
    (r'\bDELETE\s+FROM\b(?!.*WHERE)', "SQL DELETE without WHERE"),

    # Shell 注入
    (r'\b(bash|sh|zsh)\s+-[^\s]*c', "shell command via -c flag"),
    (r'\b(curl|wget).*\|\s*(ba)?sh', "pipe remote content to shell"),

    # Gateway 生命周期保护
    (r'\bhermes\s+gateway\s+(stop|restart)', "stop/restart hermes gateway"),
    (r'\bhermes\s+update\b', "hermes update"),

    # 自我终止保护
    (r'\b(pkill|killall).*\b(hermes|gateway)', "kill hermes/gateway process"),

    # Git 危险操作
    (r'\bgit\s+reset\s+--hard\b', "git reset --hard"),
    (r'\bgit\s+push\b.*--force', "git force push"),
    (r'\bgit\s+clean\b.*-f', "git clean with force"),
]
```

#### 6.4.3 审批模式

```yaml
# config.yaml
approvals:
  mode: "manual"    # manual | smart | off
  timeout: 60        # 审批超时秒数
  cron_mode: "deny"  # cron 会话模式: deny | approve
```

| 模式 | 说明 |
|------|------|
| `manual` | 危险命令需要用户确认 |
| `smart` | 使用辅助 LLM 评估风险，自动批准安全命令 |
| `off` | 关闭审批（不推荐） |

#### 6.4.4 Smart Approval 决策

```bash
命令输入 → LLM 评估风险
                │
        ┌───────┼───────┐
        ▼       ▼       ▼
    APPROVE   DENY   ESCALATE
    (安全)   (危险)  (不确定)
```

#### 6.4.5 审批结果结构

```python
{
    "approved": bool,           # 是否批准
    "message": str | None,       # 错误/提示消息
    "pattern_key": str,          # 匹配的模式键
    "description": str,          # 人类可读描述
    "status": "approval_required",  # 等待用户审批
    "user_approved": bool,       # 用户明确批准
    "smart_approved": bool,      # LLM 智能批准
}
```

#### 6.4.6 会话管理

```python
# 批准作用域
approve_session(session_key, pattern_key)   # 会话级批准
approve_permanent(pattern_key)              # 永久批准 (config.yaml)
is_approved(session_key, pattern_key)       # 检查批准状态

# YOLO 绕过
enable_session_yolo(session_key)           # 会话级绕过所有检查
```

#### 6.4.7 交互式审批

```bash
⚠️  DANGEROUS COMMAND: recursive delete
    command: rm -rf /tmp/cache

    [o]nce  |  [s]ession  |  [a]lways  |  [d]eny
```

| 选项 | 说明 |
|------|------|
| `o` (once) | 这次批准 |
| `s` (session) | 本会话批准 |
| `a` (always) | 永久批准 |
| `d` (deny) | 拒绝执行 |

#### 6.4.8 配置示例

```yaml
# config.yaml
security:
  approval:
    enabled: true
    mode: smart
    timeout: 60
    cron_mode: deny  # 定时任务拒绝危险命令

  # 永久白名单
  command_allowlist:
    - "git reset --hard"
    - "rm -rf node_modules"

  # 危险模式
  dangerous_patterns:
    - ".*rm\\s+-rf.*"
    - ".*curl.*\\|.*sh.*"
    - ".*>\\s*/dev/sd[a-z].*"
```
