## 4.7 本章小结

### 关键结论

1. **统一架构**：Gateway 通过平台适配器模式，将 17+ 消息平台收敛到同一个 `MessageEvent → AIAgent → SendResult` 流程。添加新平台只需实现 `connect`、`disconnect`、`send` 三个方法。

2. **会话隔离 vs 记忆共享**：每个 `platform:chat_type:chat_id` 组合有独立的会话上下文，但所有平台共享同一套 Memory 和 Skills。这保证了对话独立性的同时，让 Agent 的学习成果跨平台生效。

3. **企业平台优先走 QR 扫码**：钉钉和企业微信都支持 QR 扫码认证（`hermes setup dingtalk` / `hermes setup wecom`），一行命令完成接入，避免了在开放平台手动创建应用的繁琐流程。

4. **Proxy Mode 解决部署冲突**：当你需要 E2E 加密（Matrix）但又需要 Agent 访问本地文件时，Proxy Mode 让 Gateway 和 Agent 分进程运行，通过 SSE 流式通信。

5. **运维自动化**：`hermes gateway start/stop` 自动管理 systemd/launchd 服务，`hermes update` 自动重启网关，Session 自动清理防止数据膨胀。
