## 4.3 企业平台

企业 IM 平台的接入比 Telegram/Discord 复杂，涉及企业认证、应用审批和特定的协议。本节覆盖三个国内企业平台。

---

### 4.3.1 飞书 (Feishu/Lark)

飞书通过 Stream 协议连接，不需要公网回调地址。

#### 第一步：创建飞书应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 在"权限管理"中开通：
   - `im:message` — 接收消息
   - `im:message:send_as_bot` — 以机器人身份发送消息
4. 在"事件订阅"中选择 Stream 模式，订阅 `im.message.receive_v1`
5. 记下 **App ID** 和 **App Secret**

#### 第二步：配置 Hermes

```yaml
# ~/.hermes/config.yaml
feishu:
  app_id: "cli_xxxxxxxxxxxx"
  app_secret: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

或环境变量：

```bash
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

#### 第三步：启动

```bash
hermes gateway run
```

飞书使用 Stream 协议长连接，不需要配置公网 URL 或回调地址。

#### 飞书特有功能

**@mention 上下文保留**（v0.10.0+）：入站消息保留 @mention 信息，Agent 能看到完整的群消息上下文。

**流式编辑换行修复**（v0.10.0+）：流式输出不再前置多余空行，保证飞书中消息格式正确。

**飞书文档支持**：Agent 可以通过飞书 API 读写飞书文档（需要在技能中启用 `feishu_doc` toolset）。

#### 排障

| 问题 | 检查 |
|------|------|
| 应用审核中 | 企业自建应用需要管理员审批 |
| 收不到消息 | 确认事件订阅已启用 Stream 模式；确认权限已开通 |
| 消息格式错乱 | 确认 Hermes 版本 >= v0.10.0（修复了流式换行问题） |

### 4.3.2 钉钉 (DingTalk)

钉钉支持两种接入方式：Stream 协议（推荐）和 OAPI Webhook。

#### 方式一：QR 扫码认证（推荐，v0.10.0+）

Hermes 提供了 Device Flow 认证，无需手动在钉钉开放平台创建应用：

```bash
hermes setup dingtalk
```

终端会显示一个 QR 码，用钉钉手机端扫码，Hermes 自动获取 AppKey/AppSecret 并完成配置。

这是最简单的接入方式——一行命令，扫码即可。

#### 方式二：手动创建应用

1. 登录 [钉钉开放平台](https://open-dev.dingtalk.com/)
2. 创建应用 → 获取 AgentId、AppKey、AppSecret
3. 开启 Stream 模式
4. 配置 Hermes：

```yaml
# ~/.hermes/config.yaml
dingtalk:
  client_id: "dingxxxxxxxxxxxxxx"
  client_secret: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

#### 权限控制

钉钉平台支持与 Telegram/Discord 对齐的权限控制：

```yaml
dingtalk:
  require_mention: true           # 群聊中需要 @bot 才响应
  allowed_users: ["user_id_1"]    # 允许的用户列表
```

#### 技术要求

- 钉钉 Stream SDK `dingtalk-stream` >= 0.24
- 支持 Stream 协议和 OAPI webhooks 两种连接方式

### 4.3.3 企业微信 / 微信

#### 企业微信 (WeCom)

企业微信支持 Stream 协议和 QR 扫码认证：

```bash
hermes setup wecom
```

扫码获取 bot 凭证，无需手动配置。也可以手动配置：

```yaml
# ~/.hermes/config.yaml
wecom:
  corp_id: "ww..."
  agent_id: "1000002"
  secret: "..."
```

企业微信的 setup 向导（`hermes_cli/gateway.py:_setup_wecom`）通过 QR 扫码获取 bot 凭证，自动写入配置。

#### 微信 (WeChat/个人微信)

个人微信接入通过 iLink Bot API 实现，采用长轮询收消息 + AES-128-ECB 媒体加密：

> **注意**：个人微信接入使用非官方 API，存在封号风险。仅建议在测试环境使用。

配置步骤：

1. 获取微信 bot 框架（如 iLink）的凭证
2. 配置 Hermes：

```yaml
weixin:
  enabled: true
  # 具体配置项取决于 bot 框架
```

微信平台特有功能（v0.10.0+）：

- **SILK 编码语音回复**：微信的语音消息使用 SILK 编码，Hermes 自动转换
- **媒体附件收发**：图片、文件等
- **原生 Markdown 渲染**：在微信中展示格式化内容
- **CDN 白名单 SSRF 防护**：防止恶意 URL 请求内部服务

### 企业平台通用注意事项

| 维度 | 飞书 | 钉钉 | 企业微信 | 微信 |
|------|------|------|---------|------|
| 连接方式 | Stream | Stream/OAPI | Stream | 长轮询 |
| 需要公网 | 否 | 否 | 否 | 否 |
| QR 扫码认证 | 否 | 是 | 是 | 是（QR 登录） |
| 权限模型 | 应用权限 | require_mention + allowed_users | 应用权限 | 允许列表 |
| 官方 API | 是 | 是 | 是 | 非官方 |
| 风险 | 低 | 低 | 低 | 封号风险 |

**最佳实践**：

1. 优先使用 QR 扫码认证（钉钉、企业微信），减少手动配置
2. 在企业环境中，先在测试群验证 bot 行为，再推广到全公司
3. 使用 `channel_prompts` 为不同群组设置不同的系统提示和行为边界
4. 企业微信和微信的区别：企业微信走官方 API，微信走非官方 bot 框架。生产环境只推荐企业微信
