## 4.2 核心平台接入

本节覆盖最常用的四个平台：Telegram、Discord、Slack、WhatsApp。每个平台从"创建 Bot"到"发送第一条消息"的完整流程。

---

### 4.2.1 Telegram Bot

Telegram 是 Hermes 支持最完善的平台，也是推荐的首选接入平台。

#### 第一步：创建 Telegram Bot

1. 在 Telegram 中搜索 `@BotFather` 并发送 `/newbot`
2. 按提示设置 bot 名称和用户名
3. 记下返回的 **Bot Token**（格式如 `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`）

#### 第二步：配置 Hermes

```yaml
# ~/.hermes/config.yaml
telegram:
  token: "123456789:ABCdefGHIjklMNOpqrsTUVwxyz"
```

或通过环境变量：

```bash
TELEGRAM_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
```

#### 第三步：控制访问权限

默认情况下，所有人都可以和你的 bot 对话。通过允许列表限制：

```bash
# .env 或环境变量
GATEWAY_ALLOWED_USERS=telegram:123456789,telegram:987654321
```

获取你的 Telegram User ID：向 `@userinfobot` 发送任意消息即可获得。

#### 第四步：启动网关

```bash
hermes gateway run
```

在 Telegram 中找到你的 bot，发送 `/status` 确认连接成功。

#### Telegram 特有功能

**群组使用**：Bot 在群组中需要被 @mention 才会响应。建议在 BotFather 中关闭 Group Privacy（`/setprivacy` → Disable），让 bot 能看到所有消息。

**语音消息转录**：用户发送的语音消息会自动转录为文本后交给 Agent。转录使用 OpenAI Whisper（需要配置 Whisper 相关环境变量）。

**代理支持**（v0.10.0+）：如果你的服务器无法直接访问 Telegram API：

```yaml
# config.yaml
telegram:
  proxy_url: "socks5://127.0.0.1:1080"
```

或环境变量 `TELEGRAM_PROXY=socks5://127.0.0.1:1080`。

**链接预览控制**：默认情况下 Agent 发送的消息会展开链接预览。如果不需要：

```yaml
telegram:
  disable_link_preview: true
```

**贴纸支持**：用户发送贴纸时，bot 会将贴纸的 emoji 传递给 Agent。

**话题/线程**：Telegram 超级群组的论坛话题被视为独立会话，每个话题有自己的上下文。

#### 排障

| 问题 | 检查 |
|------|------|
| Bot 无响应 | 确认 `TELEGRAM_TOKEN` 正确；检查 `hermes gateway status` |
| 群组不回复 | 确认 Group Privacy 已关闭；确认 bot 是群管理员 |
| 消息截断 | Telegram 单条上限 4096 字符，Gateway 会自动分片 |
| 代理连接失败 | 确认代理 URL 格式和代理服务可用 |

### 4.2.2 Discord Bot

Discord 支持服务器频道和私聊，适合团队使用。

#### 第一步：创建 Discord Application

1. 前往 [Discord Developer Portal](https://discord.com/developers/applications)
2. 点击 "New Application" → 设置名称
3. 左侧菜单选择 "Bot" → "Add Bot"
4. 记下 **Bot Token**
5. 在 "OAuth2 → URL Generator" 中选择权限：
   - Scopes: `bot`
   - Bot Permissions: `Send Messages`, `Read Message History`, `Attach Files`, `Add Reactions`

用生成的 URL 邀请 Bot 到你的服务器。

#### 第二步：配置 Hermes

```yaml
# ~/.hermes/config.yaml
discord:
  token: "MTIzNDU2Nzg5MDEy..."
```

或环境变量 `DISCORD_TOKEN=MTIzNDU2Nzg5MDEy...`。

#### 第三步：权限控制

Discord 提供两种权限控制方式：

**用户白名单**：

```bash
DISCORD_ALLOWED_USERS=123456789,987654321
```

**角色白名单**（v0.10.0+）：允许拥有特定角色的用户使用 bot：

```bash
DISCORD_ALLOWED_ROLES=111111111,222222222
```

两种方式是 **OR 关系**——用户 ID 或角色任一匹配即放行。如果两个都没配置，所有人都可以使用 bot。

#### 第四步：channel_prompts（按频道定制提示）

不同频道可以注入不同的系统提示，让 bot 在不同频道表现不同角色：

```yaml
# config.yaml
channels:
  discord:
    groups:
      "1234567890":  # 频道 ID
        system_prompt_addition: "你是一个代码审查助手，专注于 Python 代码质量。"
        tools:
          allow: ["read_file", "search_files", "terminal"]
      "0987654321":
        system_prompt_addition: "你是一个友好的客服助手。"
```

> **提示**：`channel_prompts` 功能也扩展到 Telegram（群组/论坛话题）、Slack、Mattermost，配置结构类似。

#### Discord 特有功能

**Slash Commands**：Discord 原生斜杠命令会自动与 Hermes 的 `/new`、`/status` 等命令同步。

**语音频道**：支持 Opus 音频编码，可以在语音频道中播放 Agent 生成的语音。

**@everyone 和角色 Ping 屏蔽**：`allowed_mentions` 默认阻止 bot 触发全体通知，避免意外打扰。

#### 排障

| 问题 | 检查 |
|------|------|
| Bot 上线但不响应 | 确认 Bot 有 `Read Message History` 权限；确认消息没有触发 @mention 要求 |
| Slash Commands 不显示 | 确认在 Developer Portal 中注册了 commands；全局命令同步最多需要 1 小时 |
| 消息被截断 | Discord 单条上限 2000 字符，Gateway 自动分片 |

### 4.2.3 Slack Bot

Slack 适合企业团队场景，支持 Thread 对话。

#### 第一步：创建 Slack App

1. 前往 [Slack API](https://api.slack.com/apps) → "Create New App"
2. 选择 "From scratch"
3. 在 "OAuth & Permissions" 中添加 Bot Token Scopes：
   - `chat:write`、`channels:history`、`groups:history`、`im:history`、`files:write`
4. 安装 App 到 Workspace，记下 **Bot User OAuth Token**（`xoxb-...`）
5. 在 "Event Subscriptions" 中启用，订阅 `message.channels`、`message.groups`、`message.im`
6. 设置 Request URL（指向你的 Hermes 网关地址）

#### 第二步：配置 Hermes

```yaml
# ~/.hermes/config.yaml
slack:
  bot_token: "xoxb-xxxx-xxxx-xxxx"
  app_token: "xapp-xxxx-xxxx-xxxx"  # 用于 Socket Mode
```

或环境变量 `SLACK_BOT_TOKEN=xoxb-xxxx`、`SLACK_APP_TOKEN=xapp-xxxx`。

#### 第三步：使用 Socket Mode（推荐）

Socket Mode 不需要公网地址，适合本地开发和内网部署：

```yaml
slack:
  socket_mode: true
```

#### Slack 特有功能

**Thread 支持**：在 Slack Thread 中的对话被视为独立会话。

**Reaction 生命周期**（v0.10.0+）：通过 `SLACK_REACTIONS` 环境变量控制 bot 收发消息时的 emoji 反应。

```bash
SLACK_REACTIONS=1  # 启用
```

### 4.2.4 WhatsApp

WhatsApp 接入通过 Bridge 模式实现，需要一个 Node.js 中间层。

#### 前置条件

- Node.js 18+
- WhatsApp Bridge（Hermes 提供的 `whatsapp-bridge/` 目录）

#### 第一步：启动 WhatsApp Bridge

```bash
cd whatsapp-bridge
npm install
npm start
```

首次启动会在终端显示 QR 码，用 WhatsApp 手机端扫码关联。

#### 第二步：配置 Hermes

```yaml
# ~/.hermes/config.yaml
whatsapp:
  enabled: true
```

#### 第三步：控制权限

```bash
GATEWAY_ALLOWED_USERS=whatsapp:1234567890@s.whatsapp.net
```

#### WhatsApp 特性

- 群消息需要配置前缀触发（如 `@bot` 或 `!hermes`）
- 允许列表控制谁可以触发 bot
- 支持媒体消息收发（图片、文档）

> **注意**：WhatsApp Bridge 使用非官方 API，存在被封号风险。建议使用独立号码，不要用主号码。
